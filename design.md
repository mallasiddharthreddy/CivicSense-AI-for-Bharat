# Design Document

## 1. System Overview

CivicSense is a first-mile waste verification and logistics orchestration platform that fundamentally reimagines waste collection by placing AI-powered quality control at the center of the process. Unlike traditional waste management systems that collect first and sort later, CivicSense ensures segregation compliance at the source before any physical collection occurs.

The platform operates on a Zero-UI philosophy, leveraging WhatsApp as the universal interface to eliminate adoption barriers. Community Participants interact exclusively through familiar messaging, while an advanced AI Vision model acts as a strict gatekeeper that verifies waste quality and authorizes pickup logistics only for properly segregated materials. This approach creates a clean recycling supply chain from the very first mile.

The system orchestrates the complete end-to-end flow: Community Participant submission → AI visual reasoning → logistics activation → collection → credit allocation → community impact tracking. By positioning AI verification before logistics rather than after collection, CivicSense prevents contaminated waste from entering the recycling stream and drives behavioral change through immediate feedback and behavioral reinforcement mechanisms.

This design reflects lessons learned from an early working prototype that validated the core workflow feasibility, with the current phase focused on pilot readiness, robustness, and evaluation at community scale.

## 2. Primary Actors and Interfaces

### Community Participants (WhatsApp Interface)
Community Participants are the primary users who submit waste for verification through WhatsApp messaging. They never install applications or learn new interfaces, instead using familiar messaging patterns to photograph and submit dry waste. The WhatsApp interface handles full bilingual communication (English and standard Devanagari Hindi), provides immediate feedback on verification results, and delivers pickup confirmations and credit notifications. Community Participants receive educational guidance when submissions are rejected.

### AI Gatekeeper (Verification Engine)
Powered by Amazon Bedrock (Nova Pro), the AI Gatekeeper serves as the critical quality control checkpoint that analyzes submitted waste images. It operates on a "conservative inspector" mandate—evaluating segregation quality, identifying contamination (e.g., organic/wet waste), and making a binary eligibility decision for pickup authorization. The AI feeds educational content back to Community Participants. Crucially, it operates as a gatekeeper before logistics activation, not as a post-collection classifier.

### Drivers (Web-Based Pickup Interface)
Drivers access a web-based interface that displays only verified pickup requests generated by successful AI approvals. They view prioritized pickup lists with community participant contact information, waste descriptions, and location details. During collection, drivers record actual waste weights in kilograms through simple forms, triggering automatic credit calculations and community participant WhatsApp notifications. 

### Community Admins (Analytics Dashboard)
Community administrators access comprehensive analytics through a web dashboard that displays aggregated community performance, environmental impact metrics (total kg recycled, CO2 offset), and a live activity feed. They can also use this dashboard to trigger automated Friday Reminders, broadcasting an automated WhatsApp message to all registered users to coordinate weekend pickups.

## 3. End-to-End Workflow

### Registration and Onboarding
New Community Participants initiate contact by messaging the CivicSense WhatsApp number, providing their Villa Number. The system captures this information in DynamoDB without requiring complex registration processes.

### Waste Submission and Verification
Community Participants photograph dry waste and send images via WhatsApp. The image is uploaded to Amazon S3 for permanent storage and auditability, and the AI Gatekeeper (Bedrock) initiates analysis to determine segregation quality. The system evaluates whether all visible items are properly segregated dry waste, checking strictly for contamination, mixing, or wet materials.

### Decision Points and Approval Flow
For submissions meeting segregation standards, the AI Gatekeeper approves pickup and automatically generates a `PENDING` request in the driver interface. Approved Community Participants receive a bilingual confirmation message with expected pickup timeframes.

### Rejection and Education Flow
When mixed waste, contamination, or wet materials are detected, the AI Gatekeeper rejects submissions and provides specific bilingual improvement guidance. Rejection messages explain identified issues (e.g., fruits, vegetables, food scraps) and offer corrective advice.

### Logistics Activation and Collection
Approved pickup requests appear in the driver interface with community participant details. Drivers input the collected weight, marking the pickup as `COMPLETED`.

### Credit Allocation and Recognition
The system allocates Green_Credits to community participants based on collected weight (10 credits per 1 kg of verified dry waste) and updates community leaderboards in DynamoDB. A final WhatsApp receipt is sent to the user detailing the weight collected and points earned.

### Community Analytics Update
All interactions feed into community-level analytics, updating segregation compliance rates, environmental impact metrics, and participation statistics on the admin dashboard while maintaining individual privacy.

## 4. AI Gatekeeper Design

### Quality Control Positioning
The AI Gatekeeper operates as a pre-logistics quality control layer, making binary eligibility decisions before any physical collection occurs. This positioning prevents contaminated waste from entering recycling streams and ensures drivers collect only verified materials.

### The "Conservative Inspector" Prompt Logic
The AI Gatekeeper utilizes Amazon Bedrock (Nova Pro) operating with strict negative constraints. It is instructed to reject any image containing organic or wet waste (fruits, vegetables, peels, moisture, food residue). Even if a bag is 90% dry recyclables, the presence of any wet waste triggers an `INVALID` response.

### Classification States
* **VALID**: Return this ONLY if the image exclusively contains clean dry recyclables (plastic, cardboard, metal, glass) with zero organic matter.
* **INVALID**: Return this if the image contains ANY organic/wet waste, food scraps, or mixed bags.
* **UNCLEAR**: Return this if the bag is closed/opaque, the photo is blurry, or no waste is visible.

### Behavioral Change Integration
Rejection decisions include specific explanations of identified issues. The bilingual responses provide actionable guidance for improvement, directly addressing common segregation mistakes across the community.

## 5. Data and Event Flow (Conceptual)

### Event-Driven Architecture
The system operates on an event-driven architecture where each stage triggers subsequent processes through discrete events. Image submission events initiate AI verification, approval events generate pickup requests, collection events trigger credit allocation, and all events feed analytics updates.

### Verification and Logistics Separation
Clear separation exists between verification events (AI analysis, approval/rejection decisions) and logistics events (pickup creation, driver assignment, collection completion). This separation ensures quality control independence from operational pressures.

### Traceability and Privacy Balance
The system maintains complete traceability for compliance and dispute resolution through internal event logs (S3 image storage and DynamoDB records) while anonymizing data for public analytics. 

## 6. User Experience Philosophy

### Zero-UI Principle
The Zero-UI philosophy eliminates interface learning curves by leveraging WhatsApp's universal familiarity. Community Participants complete entire verification cycles through natural messaging patterns without downloading applications or navigating complex interfaces.

### Minimal Cognitive Load
Each verification cycle requires no more than 2 user actions: image submission and optional clarification. The bilingual setup ensures instructions are universally understood across Indian community demographics.

### Gamification as Behavior Reinforcement
Credit-based gamification provides immediate positive reinforcement for proper segregation. The 10 points per kg formula is simple and transparent, reinforcing the value of the user's effort.

## 7. Scalability and Reliability Considerations

### Compute Scalability and Timeout Management
The core compute layer (AWS Lambda) handles message parsing and coordinates the Bedrock AI calls. It is explicitly configured with a 30-second timeout to allow the Nova Pro vision model sufficient time for complex visual reasoning without dropping requests.

### Stateless Processing Architecture
Core verification and logistics processes operate statelessly where possible, enabling efficient scaling and fault recovery. State is maintained in DynamoDB for user context and credit tracking.

### Data Consistency and Backup
All interaction data, pickup records, and credit transactions are stored with strong consistency guarantees in DynamoDB. Images are permanently archived in Amazon S3 for system audits.

## 8. Cloud and Deployment Strategy (AWS Native)

### Cloud-Native, Serverless-First Approach
The platform leverages a fully serverless architecture for cost optimization, rapid prototyping, and high scalability. Functions scale independently based on demand, with managed services handling infrastructure complexity.

### Managed AI Services Integration
AI verification utilizes Amazon Bedrock's Nova Pro model, reducing operational overhead while maintaining state-of-the-art multimodal reasoning accuracy.

### Web Dashboard Hosting
Community dashboards and driver interfaces are hosted on scalable Amazon S3 static web hosting infrastructure.

## 9. AWS Architecture (As-Built Architecture)

This section describes the deployed AWS architecture supporting the pilot demonstration and community-scale deployment, emphasizing clarity, accessibility, and responsible AI use for public impact.

**AWS Components:**
* **WhatsApp Business API (Twilio Sandbox):** External messaging interface where community participants send waste images and receive verification results without app installation.
* **Amazon API Gateway:** Exposes the backend via a Greedy Proxy integration (`/{proxy+}`).
* **Compute & Orchestration Block:**
  * **AWS Lambda (FastAPI via Mangum):** Serverless compute that processes incoming requests, handles message routing, and coordinates workflow logic for the civic system.
* **AI Quality Gate (Segregation Verification) Block:**
  * **Amazon Bedrock (Nova Pro):** Performs multimodal reasoning on submitted waste images to verify segregation quality, acting as the strict quality control checkpoint before logistics are triggered.
* **Data & State Management Block:**
  * **Amazon DynamoDB:** NoSQL database storing user data (`CivicSense_Users`), pickup logs (`CivicSense_Pickups`), and credits.
  * **Amazon S3 (`civicsense-waste-images`):** Permanent object store for submitted waste images to guarantee auditability.
* **Dashboards & Interfaces Block:**
  * **Amazon S3 (Static Web Assets):** Hosts the web-based Driver Interface and Community Admin Dashboard.
* **Observability & Management Block:**
  * **Amazon CloudWatch:** Collects logs, metrics, and traces for system monitoring.

**Data Flow:**
1. WhatsApp Business API → Amazon API Gateway
2. Amazon API Gateway → AWS Lambda (FastAPI/Mangum)
3. AWS Lambda → Amazon S3 (Uploads image for storage)
4. AWS Lambda → Amazon Bedrock (Invokes Nova Pro for reasoning)
5. Amazon Bedrock → AWS Lambda (Returns VALID/INVALID/UNCLEAR)
6. AWS Lambda → Amazon DynamoDB (Updates state/logs pickup)
7. AWS Lambda → WhatsApp API (Sends bilingual reply to user)
8. Amazon S3 (Static Web Assets) → Internet (Serves Dashboard/Driver UI)

## 10. Extensibility and Future Enhancements

### Workflow Orchestration Upgrade
Transitioning the current flat Lambda architecture into formal AWS Step Functions to provide decoupled approval/rejection workflow orchestration for enterprise-grade audit compliance.

### Multi-Community Onboarding
The platform architecture supports expansion to multiple gated communities through tenant isolation and community-specific configuration management. Each community maintains independent analytics.

### Authentication and Access Control
Integrating Amazon Cognito to provide secure authentication and role-based access for community admins and collection drivers.

### Advanced Analytics and Insights
Enhanced analytics capabilities including predictive modeling for waste generation patterns, optimization algorithms for pickup routing, and advanced ESG impact measurements.
